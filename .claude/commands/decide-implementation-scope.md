# /decide-implementation-scope

ラフモック確認後に、本格実装するか、運用・教育・ルールで対応するかを判断する。

## 目的

システムを作ることを目的化せず、会社にとって必要な範囲だけを決める。

## 実行内容

1. `work/{company}/00_company/company-brain.md` を読む。
2. `work/{company}/02_actions/action-board.md` を読む。
3. `work/{company}/05_mockup/rough/review-log.md` を読む。
4. 以下を判断する。
   - 本格実装するか
   - ラフモックを修正するか
   - システム化しないか
   - 運用・教育・ルールで対応するか
5. 判断理由を `implementation-decision.md` に記録する。
6. 学んだことを `learning-log.md` に戻す。

## 成果物

- `work/{company}/05_mockup/rough/implementation-decision.md`
- `work/{company}/02_actions/action-board.md`
- 必要に応じて `work/{company}/01_learning/learning-log.md`
