# /review-rough-mockup

作成したラフモックを、会社の課題解決に役立つかという視点でレビューする。

## 目的

見た目ではなく、経営効果・現場運用・継続性を確認する。

## 実行内容

1. `work/{company}/05_mockup/rough/` を確認する。
2. `company-brain.md` と `action-board.md` に照らして、今回の課題解決に合っているかを見る。
3. 以下を確認する。
   - どの業務が楽になるか
   - 誰が使うか
   - 現場が使い続けられるか
   - 経営者が判断しやすくなるか
   - 作らない方がよい機能はないか
4. 指摘事項を `review-log.md` に記録する。
5. 学んだことを `learning-log.md` に戻す。

## 成果物

- `work/{company}/05_mockup/rough/review-log.md`
- 必要に応じて `work/{company}/01_learning/learning-log.md`
