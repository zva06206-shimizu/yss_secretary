# /create-rough-system-mockup

会社の課題を解決する打ち手としてシステム化が必要そうな場合に、まずラフモックを作る。

## 目的

いきなり本格開発せず、画面と業務フローを見える化して、作る価値があるかを判断する。

## 実行内容

1. `work/{company}/00_company/company-brain.md` を読む。
2. `work/{company}/02_actions/action-board.md` を読み、今回の目的を確認する。
3. システム化で解くべき課題か、運用・教育・ルールで解くべき課題かを確認する。
4. システム化が妥当な場合だけ、`06-build-html-mockup` スキルでラフモックを作る。本格実装に進む段階になったら `system-builder` に引き継ぐ。
5. `work/{company}/05_mockup/rough/` にラフモックを作る。
6. 作成後、学んだことを `learning-log.md` に戻す。

## 成果物

- `work/{company}/05_mockup/rough/index.html`
- `work/{company}/05_mockup/rough/mockup-brief.md`
- `work/{company}/05_mockup/rough/screen-list.md`
- `work/{company}/05_mockup/rough/assumption-log.md`
- 必要に応じて `work/{company}/01_learning/learning-log.md`
