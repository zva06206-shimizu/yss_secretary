# Problem Analysis Template

## 目的

顧客の現状を、現場課題と経営課題へ整理する。

## 構成

```md
# 現状理解

## 確認済みの事実

- {{fact_1}}
- {{fact_2}}
- {{fact_3}}

## 現時点の仮説

- {{hypothesis_1}}
- {{hypothesis_2}}

## 未確認事項

- {{unknown_1}}
- {{unknown_2}}

# 現在の課題

| 区分 | 課題 | 業務への影響 | 経営への影響 |
|---|---|---|---|
| 現場 | {{field_problem}} | {{field_impact}} | {{business_impact}} |
| 経営 | {{management_problem}} | {{operation_impact}} | {{management_impact}} |

# 放置リスク

| リスク | 発生しうる問題 | 影響する指標 |
|---|---|---|
| 属人化 | {{risk_detail}} | 教育時間、引き継ぎ時間 |
| 確認遅れ | {{risk_detail}} | リードタイム、対応漏れ |
| ミス | {{risk_detail}} | 不良率、再作業時間 |
```

## 注意

- 顧客の課題を決めつけない。
- 事実と仮説を分ける。
- 課題は必ず数字・運用・経営への影響につなげる。
