# Operation Support Template

## 目的

導入後に運用できることを示し、顧客の不安を減らす。

## 構成

```md
# 運用・保守支援

## 運用方針

{{operation_policy}}

## 支援内容

| 支援項目 | 内容 | 頻度 | 備考 |
|---|---|---|---|
| 操作説明 | {{training_detail}} | {{frequency}} | {{note}} |
| 初期設定支援 | {{setup_detail}} | {{frequency}} | {{note}} |
| 定例改善会 | {{meeting_detail}} | {{frequency}} | {{note}} |
| 問い合わせ対応 | {{support_detail}} | {{frequency}} | {{note}} |
| 軽微な修正 | {{maintenance_detail}} | {{frequency}} | {{note}} |

## 運用体制

| 役割 | 貴社 | 当社 |
|---|---|---|
| 管理者 | {{client_admin}} | {{our_admin_support}} |
| 現場利用者 | {{client_users}} | {{our_user_support}} |
| 改善判断 | {{client_decision}} | {{our_advice}} |
| 保守対応 | {{client_maintenance}} | {{our_maintenance}} |

## 定例確認項目

- 利用状況
- 未入力・未処理件数
- 現場からの問い合わせ
- 改善要望
- KPI変化
- 次月対応事項
```

## 注意

- 導入後に誰が運用するかを明記する。
- 現場が困った時の対応先を書く。
- 定例改善の仕組みを入れる。
