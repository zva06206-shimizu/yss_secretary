# Implementation Plan Template

## 目的

提案内容を、現実的に実行できる導入計画として示す。

## 構成

```md
# 導入スケジュール

## 全体方針

{{implementation_policy}}

## フェーズ別計画

| フェーズ | 期間 | 実施内容 | 貴社対応 | 当社対応 | 成果物 |
|---|---|---|---|---|---|
| 1. 要件整理 | {{period}} | {{task}} | {{client_role}} | {{our_role}} | {{deliverable}} |
| 2. 設計 | {{period}} | {{task}} | {{client_role}} | {{our_role}} | {{deliverable}} |
| 3. 構築 | {{period}} | {{task}} | {{client_role}} | {{our_role}} | {{deliverable}} |
| 4. テスト | {{period}} | {{task}} | {{client_role}} | {{our_role}} | {{deliverable}} |
| 5. 運用開始 | {{period}} | {{task}} | {{client_role}} | {{our_role}} | {{deliverable}} |

## 進行管理

- 定例会：{{meeting_frequency}}
- 課題管理：{{issue_management_method}}
- 進捗共有：{{progress_report_method}}
- 意思決定者：{{decision_maker}}

## 導入時の注意点

- {{risk_1}}
- {{risk_2}}
- {{risk_3}}
```

## 注意

- スケジュールは無理に短く見せない。
- 顧客側で必要な対応を明記する。
- 成果物をフェーズごとに書く。
- 初期導入後の改善期間も含める。
