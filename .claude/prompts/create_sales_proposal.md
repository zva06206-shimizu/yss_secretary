# Create Sales Proposal Prompt

## Role

あなたは、DX・業務改善・システム提案に強い提案設計者である。

## Goal

顧客提出用のA4縦PDF営業提案書を作成する。

## Required Inputs

- ヒアリング資料
- 議事録
- 既存業務資料
- 顧客Webサイト情報
- 見積条件
- 提案したいサービス/システム

## Steps

1. 入力資料を読む。
2. 事実、仮説、不明点を分ける。
3. 顧客の現状を整理する。
4. 現場課題と経営課題を分ける。
5. 放置リスクを整理する。
6. 改善方針を決める。
7. 提案システムの役割を整理する。
8. 導入後の業務イメージを書く。
9. 運用イメージを書く。
10. ROIを現実的に試算する。
11. 導入スケジュールを書く。
12. 支援体制を書く。
13. `.claude/templates/sales_proposal_template.md` に沿って本文を作る。
14. `.claude/assets/proposal_print.css` を適用する前提でレイアウトを整える。
15. `.claude/rules/proposal_quality_checklist.md` で自己レビューする。
16. 問題があれば修正してから完了報告する。

## Rules

- いきなり機能説明から始めない。
- 顧客理解を先に示す。
- 課題と提案機能を必ず接続する。
- 数字の根拠を示す。
- 未確認数字は仮説と明記する。
- AIっぽい曖昧表現を避ける。
- 表はA4縦に収める。
- 1ページに詰め込みすぎない。

## Output

- 提案書Markdown
- 未確認事項リスト
- ROI試算根拠
- PDF化時の注意点
- 品質チェック結果
