# Create Test Design

## 目的

要件、画面、機能に対してテストケースを作成し、実装前に抜け漏れを確認する。

## 手順

1. `03_design/screen-function-matrix.md` を読む。
2. 要件ID、画面ID、機能IDごとにテストケースを作る。
3. 正常系、異常系、境界値、権限、セキュリティ、アクセシビリティを含める。
4. `04_test/test-design.md` と `04_test/test-cases/` を更新する。
5. 不足があれば `08_issues/backlog.md` に記録する。

## 完了条件

- 実装対象に対応するテストケースが存在する。
- 未テストの機能が一覧化されている。
