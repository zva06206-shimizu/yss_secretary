# Implement Laravel

## 目的

HTMLモック、画面設計、機能設計、テストケースに基づき、Laravel + MySQL（本リポジトリの既定スタック。要件に応じて他スタックへ変更可）で実装する。

## 手順

1. 対象の要件ID、画面ID、機能ID、テストケースIDを確認する。
2. HTMLモックをBladeへ展開する。
3. Controller、FormRequest、Service、Model、Policy、Migrationを分離して実装する。
4. 認証・認可・入力検証・エスケープ・CSRFを確認する。
5. DB変更時はDB設計書を更新する。
6. 実装後、`screen-function-matrix.md` のLaravel実装欄を更新する。

## 完了条件

- 対象機能がローカルLaravelで動作する。
- 設計書と一覧表が更新されている。
