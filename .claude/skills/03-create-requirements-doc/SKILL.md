# Create Requirements Document

## 目的

ヒアリング内容を要件定義書として整理する。

## 手順

1. `01_research/` と `02_requirements/hearing-log.md` を読む。
2. `purpose-definition.md` を更新する。
3. `requirements-definition.md` に機能要件・対象外範囲・優先度を記録する。
4. `business-flow-as-is.md` と `business-flow-to-be.md` を更新する。
5. `non-functional-requirements.md` に可用性、性能、運用、保守、移行、セキュリティを記録する。
6. 未確定事項を `unresolved-questions.md` に残す。

## 完了条件

- 目的、業務範囲、機能要件、非機能要件、対象外、未確定事項が揃っている。
