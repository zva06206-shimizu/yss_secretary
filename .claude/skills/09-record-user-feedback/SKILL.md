# Record User Feedback

## 目的

ローカル環境での利用者確認結果を記録し、改善要望・不具合・仕様変更を管理する。

## 手順

1. 利用者、確認日、確認対象画面、確認観点を記録する。
2. 指摘内容を `07_user-review/user-feedback-log.md` に記録する。
3. 改善要望を `07_user-review/improvement-request-list.md` に整理する。
4. 不具合・仕様変更は `08_issues/issue-management.md` に起票する。
5. 設計変更が必要な場合は `00_project-control/change-log.md` に記録する。

## 完了条件

- 利用者指摘が一覧化されている。
- 対応要否、優先度、影響範囲が明確になっている。
