# Clients（顧客の長期文脈）

顧客ごとの、案件をまたいで使う長期的な文脈を残す。`client-context` エージェント／Skill `13-record-executive-context` の保存先。

案件単体の資料は `work/{project}/00_context/` に置き、ここには「次の案件でも効く」恒久的な理解だけを残す。

## ルール

- 1顧客 = 1ファイル。命名は `{client-name}.md`。
- 事実・仮説・未確認を分ける。
- 個人情報は必要最小限。機密は要約にとどめる。
- 古い情報は更新日で区別し、混ぜない。

## フォーマット

```md
# {顧客名}

- 事業・規模・体制：
- 意思決定者・キーパーソン：
- 課題・ニーズ（事実/仮説）：
- 過去の提案・取引：
- 響くポイント / 地雷：
- 未確認事項：
- 更新日：
```
