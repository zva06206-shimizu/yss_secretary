# Decision Principles

## 目的

経営者の判断軸を明文化し、Claude Codeが提案・設計・開発判断で一貫した補助をできるようにする。

## 基本判断軸

| 判断軸 | 内容 |
|---|---|
| 現場で使えるか | 実際の利用者が迷わず使えるか |
| 回り続けるか | 初期導入だけでなく、運用が継続できるか |
| 数字で説明できるか | 売上、粗利、工数、品質、教育時間などに接続できるか |
| 実装速度 | 机上の理想より、短期間で検証できるか |
| 継続性 | 属人化せず、仕組みとして残るか |
| 顧客の現実性 | 中小企業の人員、IT耐性、予算、運用力に合うか |

## 重視KPI

- 売上
- 粗利
- 稼働率
- 不良率
- リードタイム
- 教育時間
- 定着率
- 改善件数
- DX投資回収

## 提案判断

- ツール導入を目的にしない。
- AI、Webシステムは業務が回るための手段として扱う。
- 経営者には数字、現場には運用イメージで説明する。
- 最初に回る最小構成を優先する。
- 顧客が社内説明できる言葉に変換する。
