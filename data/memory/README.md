# Memory

## 目的

このディレクトリは、Claude Codeを経営者の右腕として育てるためのリポジトリ内記憶領域である。

経営相談、意思決定、顧客文脈、事業仮説、提案知見、開発知見、KPI、リスク、週次レビューを蓄積し、次回以降の提案・設計・開発・経営判断へ反映する。

## 構成

```txt
data/memory/
├── README.md
├── company/          # 会社の恒久情報・各種台帳（KPI実績/アクション/リスク/市場ウォッチ等）
├── decision-principles.md
├── consultations/    # 経営相談ログ
├── meetings/         # 会議・議事録
├── decisions/        # 意思決定ログ（理由・選択肢）
├── clients/          # 顧客の長期文脈
└── learnings/        # 横断的な学び
```

## 運用原則

- 事実、仮説、判断、次アクションを分ける。
- 古い情報は古いと分かるようにする。
- 顧客ごとの長期文脈は `clients/` に分ける。
- 案件内の具体資料は `work/{project}/` に置く。
- 意思決定は必ず理由を残す。
- 週次または案件完了時に学びを反映する。
- 提案や設計で使った知見は、再利用できる形に整理する。

## 作業開始時に見る順番

1. `decision-principles.md`
2. `company/`
3. 対象顧客がある場合は `clients/{client}.md`
4. 関連する `consultations/`
5. `decisions/`
6. 必要に応じて `learnings/`

## 注意

この領域は、会話の外にある永続的な経営メモリである。Claude Codeが同じ判断・同じ提案品質を再現するため、曖昧な記憶ではなくファイルとして残す。
