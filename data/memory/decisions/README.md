# Decisions（意思決定ログ）

「決めたこと」と「なぜそう決めたか」を再現できる形で残す。後から判断の妥当性を検証し、判断軸を磨くための領域。

`company/action-ledger.md` の Decisions 表が要点の一元台帳。ここには、理由・前提・比較した選択肢まで含む詳細を、重い意思決定について残す。Skill `16-evaluate-roi-and-cost`（投資判断）などの保存先。

## ルール

- 1意思決定 = 1ファイル。命名は `YYYY-MM-DD-{topic}.md`。
- 判断軸（`decision-principles.md`）と照らした理由を必ず残す。
- 比較した選択肢・却下した案とその理由も残す。
- 繰り返し効く判断パターンは `decision-principles.md` へ昇格する。

## フォーマット

```md
# {日付} {決定事項}

- 決めたこと：
- 背景・前提（事実/仮説）：
- 比較した選択肢：
- 採用理由（判断軸）：
- 却下した案と理由：
- 紐づくアクション（action-ledger）：
- 後で検証するタイミング：
```
