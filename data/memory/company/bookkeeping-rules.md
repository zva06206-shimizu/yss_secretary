# YSS 記帳ルール（MFクラウド会計・仕訳の型）

YSソリューションズの日々の記帳で迷わないための型。2026-06-30 整理・確定。
按分は [[expense-allocation-rules]]、資金繰りは [[cost-structure]]、消費税は [[current-issues]]（I006）参照。

## 基本方針

- **3月以降は発生主義（請求書ベース）に統一**。請求書を出したら売上を立て、入金で消し込む。
- **1〜2月（請求書発行前）は現金主義（入金時に売上）のまま据え置き**でよい（同月稼働・同月入金で期ズレなし）。
- 銀行・カードは自動連携。**連携は「動き」だけを取り込み、期首残高は取り込まない**（期首は開始仕訳で手入力）。

## 仕訳の型

| 場面 | 仕訳 |
|---|---|
| 請求書発行（売上計上） | 売掛金（取引先）/ 売上高 ＋ 仮受消費税（税込→税抜＋消費税に分離） |
| 入金（売掛の回収） | 普通預金 / 売掛金（取引先） |
| 期首売掛の回収（fanfare型） | 普通預金 / 売掛金（取引先）※売上高にしない |
| クレカ利用 | 費用 / 未払金（カード） |
| クレカ引落 | 未払金（カード）/ 普通預金 |
| 小規模企業共済・iDeCoの引落 | 事業主貸（経費でなく所得控除のため） |
| 個人資金を事業口座へ | 事業主借（※送金元も連携口座なら「口座間振替」にして二重回避） |

## 二重計上を防ぐ鉄則

- **入金を`売上高`にしない**（請求書を出した取引先は必ず`売掛金`で消込）。連携は入金を全部`売上高`と推測するので人が直す。
- 銀行連携が`普通預金/売上高`で立てた請求済み取引先の入金は **`普通預金/売掛金`に振り替える**。
- 同じ売上を「請求書側」と「銀行連携側」の両方で立てない。**売上高は1取引につき1回だけ**。

## MFクラウド請求書 → 会計 連携設定（確定値）

帳票設定 > 会計連動：
- 売上計上日：**当月末**（請求日を稼働月内に発行すれば期ズレしない）
- 請求書に対応した売掛発生の仕訳：**作成する**
- 請求書に対応した入金予定の仕訳：**作成しない**（銀行連携が実入金を記録するため。ONだと`普通預金/売掛金（未実現）`が二重に出る）
- 取引先名に応じた補助科目：**作成する**（売掛金に取引先が付かないと消込できない）

※**設定変更は「以降に発行する請求書」にしか効かない**。発行済みの過去分は手動でbackfillする。

## 期末（12/31）の宿題 ＝ 65万控除の生命線

- 青色65万控除は**発生主義が原則**（現金主義の特例は10万控除まで）。
- 発生主義（請求書ベース）に統一すれば、12月請求分（翌1月入金）も12月付で売掛が自動で立つ＝**期末の未収計上が自動**。
- 請求書を出していない売上があれば、その分だけ12/31に手動で `売掛金/売上高` を計上。

## 消費税（インボイス課税事業者）

- 売上計上で**仮受消費税が分離計上**される＝そのまま消費税申告（2割特例・要確認）の集計になる。
- 預かった消費税は別口座に取り置き（I006）。登録番号 T9810049490201。

## 検算

- **売掛金の補助元帳**：受領済み取引先は残高0、未入金月だけ残高が残る。
- **普通預金の残高**：実際の通帳残高と一致（大きく多い＝二重計上で膨張のサイン）。
