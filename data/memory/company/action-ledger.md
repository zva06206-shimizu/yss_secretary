# Action Ledger（横断アクション台帳）

## 目的

会議・経営相談・指示から生まれた「やると決めたこと」と「まだ決まっていないこと」を、案件をまたいで一元管理する。
誰が・いつまでに・どうなったか、を追える状態にして、実行の取りこぼしと属人化を防ぐ。

`current-issues.md` が「課題のスナップショット」なのに対し、ここは「実行を回す生きた台帳」である。

## 使い方

- 会議・相談・指示のたびに、生まれたアクションと未決事項をここへ追記する。会議の前後整理は Skill `15-facilitate-meetings-and-decisions` を使い、決定・ToDo・未決をここへ起票する。
- 状態が変わったら必ず「更新日」を直す。完了・中止も消さず記録として残す。
- 期限超過（状態が「進行中/未着手」で期限を過ぎたもの）は `/executive-review` で洗い出す。
- 案件固有の細部は `work/{project}/99_logs/` に置き、ここからリンクする。

状態の定義：`未着手` / `進行中` / `完了` / `保留` / `中止`

## 注意

- 事実・仮説・未確認を分ける（確度欄）。
- 機密度の高い人事・財務に関わるアクションは、内容を要約にとどめる。
- 完了・中止も履歴として残す（消さない）。
