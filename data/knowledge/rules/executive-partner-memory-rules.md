# Executive Partner Memory Rules

## 目的

Claude Codeを、単なる開発支援ではなく、経営者の右腕として継続的に強化する。

経営相談、事業判断、顧客課題、提案方針、失敗、改善、数値、意思決定をリポジトリ内に記録し、次回以降の提案・設計・開発・営業・経営判断に反映する。

## 基本方針

- 記憶はこのリポジトリ内に保存する。
- グローバル記憶や会話履歴に依存しない。
- 経営相談は、単なるメモではなく、意思決定・仮説・アクション・未解決論点に分解する。
- 経営者の判断軸、避けたいこと、重視KPI、提案スタイルを蓄積する。
- 顧客ごとの文脈、課題、過去提案、合意事項、懸念を蓄積する。
- すべての記録は、次回の提案・設計・実装で参照できる形にする。
- 推測と事実を分ける。
- 古い情報は古いと分かるようにする。

## 記録対象

| 分類 | 記録内容 |
|---|---|
| 経営相談 | 相談内容、背景、判断、論点、次アクション |
| 意思決定 | 決定事項、理由、採用/不採用案、影響範囲 |
| 経営者の判断軸 | 重視するKPI、避けたい表現、提案方針、価格感 |
| 顧客文脈 | 企業情報、課題、担当者、過去提案、懸念、進捗 |
| 事業仮説 | 仮説、検証方法、結果、次の修正 |
| 提案知見 | 刺さった表現、失敗した表現、通った提案、通らなかった提案 |
| 開発知見 | 実装方針、設計判断、失敗、改善策 |
| 業務データ知見 | KPI、改善件数、ボトルネック、現場反応 |
| リスク | 事業リスク、顧客リスク、技術リスク、運用リスク |
| 定例レビュー | 週次/月次の振り返り、改善方針、次回重点 |

## 記録ルール

### 経営相談

経営相談を受けた場合、以下を記録する。

- 相談日
- 相談テーマ
- 背景
- 事実
- 仮説
- 論点
- 選択肢
- 推奨判断
- 判断理由
- 次アクション
- 未解決事項
- 関連する `work/{project}/`

### 意思決定

意思決定は必ず理由を残す。

- なぜ決めたか
- 何を捨てたか
- どのKPIに効くか
- どのリスクがあるか
- 後から見直す条件は何か

### 顧客文脈

顧客ごとに以下を残す。

- 会社概要
- 相談背景
- 経営課題
- 現場課題
- 提案履歴
- 見積履歴
- 反応
- 懸念
- 次回提案方針

## 使い方

作業開始時に、該当する以下を確認する。

- `data/memory/decision-principles.md`
- `data/memory/company/`
- `data/memory/consultations/`
- `data/memory/clients/`
- `data/memory/decisions/`
- `data/memory/learnings/`

## 禁止

- 経営相談を単なる議事録で終わらせること
- 決定理由を残さないこと
- 顧客の懸念を消すこと
- 推測を事実として保存すること
- 古い情報を最新情報のように扱うこと
- リポジトリ外の記憶に依存すること

## 完了条件

経営相談対応後は、最低限以下が更新されていること。

- 相談ログ
- 意思決定ログ
- 次アクション
- 未解決事項
- 必要に応じて顧客文脈
- 必要に応じて提案・設計・開発ルール
