# data/

## 目的

`data/` は、案件をまたいで使う会社理解・学習・知識・テンプレート・ツールを蓄積する。

案件成果物は `data/` に置かない。案件成果物は `work/{project}/` に置く。

## 構成

| ディレクトリ | 用途 |
|---|---|
| `memory/` | 会社理解・判断軸・学習（company / learnings / meetings / clients） |
| `knowledge/` | 白書・統計・ルール・参照・出典レジストリ |
| `ui/` | UI共通部品 |
| `templates/` | テンプレート |
| `tools/` | スクリプト・ツール |
| `logs/` | 変更ログ |

## 注意

- `data/` は会社理解の蓄積層であり、案件成果物は `work/` に置く。
- 事実・仮説・未確認を分けて記録する。
- 秘密情報・APIキー・パスワード・個人情報を置かない。
