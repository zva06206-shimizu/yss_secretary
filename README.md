# 経営者の右腕AI ワークスペース

会社のことを理解し続け、経営者の右腕として考え、整理し、提案し、必要な仕事を進めるためのClaude Codeテンプレートです。

システム開発は目的ではありません。会社を理解し、課題を整理し、必要な打ち手を実行する中で、必要な場合だけシステム化します。

## トップ構成

トップディレクトリは、基本的に次の3つだけ見ればよい構成にします。

```text
.claude/   Claude Codeを動かす設定
work/      今動いている仕事・案件
data/      会社理解・学習・ナレッジの蓄積
```

## 3つの役割

### .claude/

Claude Codeの操作盤です。

例:

- agents
- skills
- hooks
- settings
- commands

ここには、仕事の中身や顧客情報を置きません。

### work/

いま動いている仕事を置く場所です。

例:

- 提案書を作る
- 議事録をまとめる
- 業務フローを作る
- 採用資料を作る
- システム構想を作る
- ラフモックを作る
- 実装を進める
- 運用改善を記録する

### data/

会社について学んだこと、次の仕事でも使う知識、テンプレート、道具を蓄積する場所です。

例:

- 会社の基本情報
- 経営者の考え方
- 強み・弱み
- 現場課題
- よく出る論点
- 判断基準
- 白書・公的統計
- 提案書や設計書のテンプレート
- UI部品
- 実行ログ

## 基本の流れ

```text
1. data を見て会社を理解する
2. work で今やる仕事を進める
3. 仕事で学んだことを data に戻す
4. 次の仕事の判断精度を上げる
```

## 初回セットアップ

初めて使うときは、Claude Codeで以下を確認します。

```txt
/initial-setup
```

初回セットアップの詳細手順は以下です。

```txt
data/templates/onboarding/initial-setup-guide.md
```

最低限、以下を実行します。

```bash
chmod +x data/tools/scripts/*.sh .claude/hooks/*.sh
./data/tools/scripts/init-company.sh company-name
```

その後、案件ごとに `work/{client-or-project}/` を作成します。

## 右腕AIがやること

- 会社の強み・弱み・課題を整理する
- 経営者の考えを言語化する
- 優先順位を決める
- 会議内容を次アクションに変える
- 提案書、議事録、業務フロー、マニュアル、営業資料を作る
- 必要ならシステムのラフモックを作る
- さらに必要なら本格実装へ進める
- 毎回のやり取りから会社理解を更新する

## 判断基準

右腕AIは、以下を基準に判断します。

- 売上が増えるか
- 粗利が改善するか
- 工数が減るか
- ミス・手戻りが減るか
- 属人化が減るか
- 現場が回りやすくなるか
- 経営判断が速くなるか
- 継続して運用できるか

## システム化の扱い

システムは打ち手の一つです。

いきなり開発せず、必要な場合だけ以下の順で進めます。

```text
課題を整理する
↓
ラフモックを作る
↓
作る価値を判断する
↓
必要な範囲だけ実装する
```

## 中核思想

AIに単発作業をさせるのではなく、会社のことを学ばせ続ける。

`work` で仕事を進め、`data` に学びを戻す。これを繰り返して、次の提案や判断が少しずつ良くなる状態を作る。
