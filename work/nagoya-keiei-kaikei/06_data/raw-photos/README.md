# 現地調査 写真の置き場（raw-photos）

ここに iPhone で撮影した現状調査の写真を置く。**画像ファイルは git 管理外**（IP・パスワード・シリアルが写るため）。このフォルダにある `.md`（索引）と、マスク済みの報告書だけをコミットする。

## 1. iPhone → PC への取り込み

- USBケーブルでPC接続 → エクスプローラ／「写真」アプリで取り込み（最も確実）。
- または iCloud写真・Googleフォト・自分宛メール/LINE でも可。
- 取り込んだ画像を**このフォルダ**に入れる。

### ⚠️ HEIC 形式に注意
- iPhone標準の **HEIC形式は読み取れないことがある**。**JPEG で書き出す**こと。
- 今後の撮影は iPhone設定「**カメラ → フォーマット → 互換性優先**」にすると JPEG で撮れる。

## 2. 命名ルール（並べ替えと索引づけのため）

`場所-カテゴリ-連番.jpg` で。例：`3f-rack-01.jpg`, `2f-device-label-04.jpg`

カテゴリの優先度（情報価値の高い順）：
1. `diagram` … 構成図・ネットワーク図（既存資料が最重要）
2. `device-label` … 機器の型番・シリアルのラベル
3. `rack` … ラック全景・設置状況
4. `doc` … 回線契約・機器台帳などの書類
5. `cabling` … 配線・ポート
6. `setting` … 設定画面 ※**パスワード/IPが写るので特に厳重**

## 3. 撮影が大量なときの捌き方

- **全部を等価に扱わない**。上の優先度1〜4を先に。ブレ・重複は捨てる。
- 索引 `../photo-index.md` に「写真ファイル → 何が写っているか」を1行ずつ記録。
- 索引ができたら Claude（クロエ）が画像を読んで、型番・年式・構成を報告書へ転記する。

## 4. セキュリティ

- 設定画面・パスワード・IP・シリアルが写った写真は機微情報。**git に上げない（本フォルダは ignore 済み）**。
- 報告書に書くときは、パスワード等は記載しない／必要なら一部マスクする。
