# 名古屋経営会計専門学校 ネットワーク現状調査 結果サマリ

> 共有先：ドットシンク（株） 御中
> 作成：YSソリューションズ（清水）
> 調査実施日：2026-06-23（一部 6/24 継続）
> 区分：本書は外部共有用サマリ（ID/PW・暗号鍵・グローバルIP等の秘匿情報は記載しない／別途厳重管理）

---

## 1. 調査の目的・範囲

- 目的：各号館の**ネットワーク現状（機器・配線・無線・セグメント・セキュリティ）を実機で把握**し、次フェーズ（機器入替・総合管理＝N-02）の判断材料を作る。
- 対象：**2号館・5号館・6号館**（1号館は当社にて再構築済みのため対象外）。
- 方法：現地で機器の目視確認、設定（config）取得、IPスキャン、配線トレース、写真記録。

---

## 2. 全体サマリ（結論）

1. **校舎は1号館をハブにしたVPNスター構成**で接続されており、各号館がそのスポークになっている。
2. **号館ごとに構成思想がバラバラ**。同じ学校内に「良い構成（2号館＝UTMインライン＋各階スター）」と「巻き直しが要る構成（5号館＝無管理スイッチのカスケード）」が混在している。
3. **共通の弱点が3つ**確認できた：
   - ① **守るべきサーバが UTM（FortiGate）の保護外に出ている**号館がある（5号館・6号館）。
   - ② **職員／生徒のネットワーク分離が実装されていない**号館がある（2号館はフラット）。
   - ③ **全棟の防犯カメラが家庭用Wi-Fiカメラ（TP-Link Tapo C110）**で、管理外・録画先が外部クラウドの可能性。
4. **UTM（FortiGate-50E）は全棟ともEOS（販売終息）世代**。更新時期。
5. 結論：**現状は「つぎはぎ」で運用されており、セキュリティ・可用性・管理性のいずれにも改善余地が大きい**。N-02（機器入替＋総合管理）で**校舎全体を統一構成へ揃える**価値が明確。

## 3. 号館別 確認結果

### 2号館（仲田／現在未使用・将来使用予定）

| 項目 | 確認結果 |
|---|---|
| 建物 | 4階建て。1F=事務、2F/3F=教室（各2室）、4F=住居型研修室（配線なし） |
| 構成 | **UTM（FortiGate）が全フロアの手前に入る「インライン構成」＋各階HUBがスター接続**。＝**理想形に近い** |
| セグメント | **単一フラット（職員／生徒の分離なし）**＝改善ポイント |
| 無線 | 各教室の天井にAP（各階2台）。**1台/階で足りる可能性**があり台数の見直し余地（要電波実測） |
| プリンタ | 3Fの複合機はネットワーク非接続（単独運用） |
| その他 | 事務デスク下に**管理外の家庭用ルータ等（持ち込み機器）**が接続されており、多重NATの状態 |

→ **2号館の「UTMインライン＋各階スター」は、他号館に展開したい目標形が同一校内に既に存在する好例**。

### 5号館（池下）

| 項目 | 確認結果 |
|---|---|
| 構成 | 職員網＝管理型スイッチの縦系／**生徒網＝無管理スイッチのカスケード（数珠つなぎ）**が物理的に並走 |
| 弱点 | **生徒網が丸ごと無管理スイッチ上**＝監視・ループ防止・品質制御なし。**一斉アクセス（オリエン等）が最も脆い側に乗る** |
| 単一障害点 | 生徒トラフィックが**1本のアップリンクに集約**＝帯域ボトルネック兼・単一障害点 |
| UTM | FortiGateは**一部のみ保護（サーバが保護外に出ている疑い）**。要最終確認 |
| ルータ | 4館で最古（2017年導入）＝経年・故障リスク |

→ **5号館は巻き直し対象**。N-02では2号館型（管理型スイッチ＋スター＋分離）へ揃える。

### 6号館（仲田／2F〜8Fの縦長）

| 項目 | 確認結果 |
|---|---|
| 構成 | **5Fを幹線分配点（コア）とした各階home-runスター**＝物理構成は良好 |
| 弱点 | **2FオフィスのサーバとAPが UTMを迂回してルータ直結＝サーバがUTMの保護外**（最重要所見） |
| 設置 | ルータ等が**2F給湯室の配電盤内**に収容。オフィスAPは**デスク上にアンテナむき出し**（天井設置でない） |
| 上階 | 7F/8Fは給湯室配電盤内に子HUB、教室にAP。電源はACアダプタ運用 |
| 残作業 | 各機器のConfig収集・フロア高計測・カメラ設定確認は**6/24に継続** |

→ **物理は良いがセキュリティ経路に穴**。サーバをUTM配下へ戻す再結線がN-02の要点。

## 4. 主要所見（横断）

1. **サーバのUTM保護外問題（5号館・6号館）**：本来UTMで守るべき業務サーバが、UTMを通らない経路に接続されている。**外部脅威・内部不正に対して無防備**になりうる。
2. **ネットワーク分離の未実装（2号館はフラット）**：職員・生徒・ゲストが同一網に同居。**生徒端末から職員・サーバ側へ到達できる構造**はリスク。
3. **全棟カメラが家庭用Wi-Fiカメラ（TP-Link Tapo C110）**：
   - 専用録画機（NVR）がなく、**録画が外部クラウド（海外事業者）に出る可能性**。
   - **学校が生徒を撮影する映像のプライバシー管理上、看過できない**。
   - 管理外・校内Wi-Fiに相乗りで帯域も消費。
   - → N-02で**業務用カメラ＋ローカル録画（NVR）＋アクセス制御＋専用セグメント**への置換を推奨。
4. **UTM（FortiGate-50E）が全棟EOS世代**：サポート終息＝脆弱性対応・更新が受けられない。更新必須。
5. **持ち込み・管理外機器（シャドーIT）**：家庭用ルータ／AP等が現場判断で接続され、多重NAT・管理不能の温床。
6. **設計資料と実態の差分**：旧資料に載る機器（一部AP・サーバ等）が実機に無い／別機種、という差分を多数確認。**「現状を正しく把握する」こと自体が今回の調査の価値**。

---

## 5. 次フェーズ（N-02）への論点

- **統一構成への移行**：全号館を「管理型スイッチ＋タグVLANによる職員/生徒/ゲスト分離＋UTMインライン＋各階スター」へ統一（＝2号館・1号館再構築版の形）。
- **サーバをUTM配下へ**：迂回経路の是正（再結線）。
- **無線の適正化**：天井設置・台数最適化・管理型APでの一元管理。
- **カメラの業務用化**：ローカルNVR・アクセス制御・専用セグメント。
- **UTM更新**：EOS世代からの入替＋ライセンス・運用主体の整理。
- **総合管理（運用）**：機器の一元監視・構成図の整備（現状は図面が古く実態と乖離）。

## 6. 残作業（6/24 継続）

- 6号館：各機器のConfig収集、各階フロア高の計測、カメラ（録画先・設定）の確認、一部経路の最終確認。
- 各棟：UTMのライセンス状況・運用主体（管理ベンダー）の確認は学校経由で照会予定。
