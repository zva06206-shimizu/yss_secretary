# 現地写真「どこだっけ問題」をなくす撮影方式（6/23）

> 課題：撮って持ち帰ってから見返すと「この機器どこの何だっけ？」になる。
> 原因：機器のアップだけ撮ると、場所情報が写真に残らない／順番も分からなくなる。
> 解決の核：**場所ごとに必ず「見出し写真」を1枚目に撮る**＋**決まった順で撮る**＋**撮影ログで時刻と紐付ける**。これだけで後から場所不明がほぼ消える。

## 推奨方式（ワンオペ前提・iPhone）

### ルール1：場所が変わったら「見出し写真」から（最重要）
新しい部屋・配電盤・ラックに着いたら、機器を撮る前に**見出しを1枚**：
- **スレートカード**（下記テンプレを印刷 or 小型ホワイトボード）に「館・階・部屋・対象・通し番号」を書いて、その場で1枚撮る。映画のカチンコと同じ。
- カードが無い時の代用＝**部屋のドア札/部屋番号プレート/配電盤の扉**を1枚。
→ 見返す時、この見出し写真が「ここから下は◯◯館◯階」の区切りになる。

### ルール2：1か所で必ず同じ順に撮る（4カット固定）
1. **引き**（部屋全景・配電盤を開けた全体）＝位置関係が分かる
2. **機器全景**（ラック/盤の中の機器のまとまり）
3. **型番ラベルのアップ**（メーカ/型番/S/N/MAC）＝機種特定
4. **ポート・配線**（どのポートに何が刺さっているか、ケーブル色）
→ 順番が固定だと、時系列で見たとき「引き」が出るたびに新しい機器、と読める。

### ルール3：付箋で場所IDを写し込む（一石二鳥）
機器に**付箋**へ機器ID（例 `6-5F-CORE`／`5-2F-SW`＝館-階-種別）を書いて貼り、その状態で撮る。
- 写真の中に場所IDが必ず写る＝後から確実に判別。
- そのままにしておけば**A-2のラベリング工事の下地**になる（無駄にならない）。

### ルール4：撮影ログを1行ずつ（時刻が命綱）
下記ログ用紙に「時刻・館階部屋・対象・メモ」を手書き。
iPhoneのEXIFに撮影時刻が残るので、**帰宅後はEXIF時刻順に並べ、ログと突合**すれば順番が崩れない。

### 補助
- iPhoneの**写真キャプション**（編集→キャプション追加）か**ボイスメモ**で「6号館5Fコアスイッチ」と一言入れる。
- 暗い盤内は**懐中電灯/フラッシュ**。型番が反射で読めない時は角度を変えて2枚。

## 帰宅後の整理フロー
1. 写真をPCへ取り込み、**撮影時刻（EXIF）順**に並べる。
2. 「見出し写真」で塊を区切り、各塊の場所を確定。
3. 計画書§4の命名規則 **`館-カテゴリ-番号`**（例 `6-rack-01`、カテゴリ＝diagram/device-label/rack/cabling/wifi/server/camera/doc）でリネーム。
4. `06_data/photo-index.md` に1行ずつ記録。

## 印刷用①：スレートカード（各場所でこれを持って1枚撮る）

> A5で数枚印刷し、油性ペンで都度書き換える。または小型ホワイトボードで代用。

```text
┌───────────────────────────────┐
│  名古屋経営会計専門学校 NW調査 2026-06-23 │
│                                           │
│  館 ：（ 2号館 / 5号館 / 6号館 ）          │
│  階 ：______ 階                            │
│  場所：__________________（部屋/配電盤/ラック）│
│  対象：__________________（機器名）         │
│  No ：__________                          │
└───────────────────────────────┘
```

## さらに楽にしたい場合（任意・次回以降）
- **ラベルプリンタ**（テプラ等）でその場で機器IDシールを貼る＝付箋より恒久的・A-2工事と直結。
- 位置情報ONで撮ると館の区別はGPSでも付くが、屋内・階の特定は不可＝**見出し写真が本命**。
