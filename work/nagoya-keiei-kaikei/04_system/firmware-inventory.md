# 名古屋経営会計専門学校 NW機器 ファームウェア／EOL 棚卸し（N-02提案根拠）

> 用途：N-02提案に「**ファームウェア未更新・EOL（サポート終了）リスク**」を数値根拠で載せるための棚卸し表。
> ※本ファイルは git 管理対象。FW版・型番・稼働日数・IPは記載可。**S/N・MAC は →credentials**（ここには書かない）。
> ※「最新FW」「EOL」列は Buffalo/ELECOM/Fortinet/YAMAHA 公式の要確認（オフラインのため後追い／WebでYutaka承認後に確定）。

## なぜリスクか（提案の論点）

- **既知脆弱性の放置**：数年前のFWは公表済み脆弱性が未修正のまま＝侵入・踏み台・情報漏洩の入口。学校＝生徒個人情報を扱う。
- **EOL/EOS（サポート終了）**：メーカー保守・セキュリティ更新・部品供給が切れる。故障時に同等品調達不可＝授業停止リスク。
- **更新運用の不在**：「稼働○百日・無停止」「GW未設定で外部到達なし」「管理者未定義」＝**誰も面倒を見ていない**状態の客観証拠。
- **無管理スイッチ**：そもそもFW更新UIが無い民生機（ELECOM EHB/EHC系）＝**更新不能＝恒久的に塩漬け**。

## 既に分かっている強い材料

1. **FortiGate 50E＝EOS世代**（5号館はFG-40F）。**ライセンス切れならUTMは素通り＝最も訴求力の高い更新根拠**。管理ベンダー＝シャープMJ経由でライセンス期限を要照会。
2. **BS-GS2016P＝2019年FW・326日無停止・GW未設定・管理者未定義**＝「更新運用ゼロ」の客観証拠。
3. **RTX1210＝2016年FW**（6号館）。RTXは無償FW更新が出ている世代＝放置の証拠。
