# Zoho Invoice テンプレ設計メモ（試作）

最終更新: 2026-06-25 ／ ※澤田さんの実データ受領前の仮設計。資料が来たら数値・習い事名を差し替える。

## 設計判断

- **2個目の習い事10%割引＝「行レベル割引（At Line Item Level）」** で持つ。2個目の月謝行にだけ10% off。直感的・毎月編集が楽。代替（マイナス品目）より素直。
- 「4種テンプレをコピー」の正体 ＝ **①PDF見た目テンプレ（英語・1〜2種）** ＋ **②内容のひな型を Clone / Recurring で使い回す**。
- 固定額の生徒＝Recurring Invoice（自動）／変動の生徒＝ひな型をCloneして毎月編集（半自動）。

## 初期設定

1. **通貨**：日本サインアップだと基準通貨をSGDに変更できない（base候補に出ない＝Zoho仕様）。
   - **回避策＝基準はJPYのままでOK**。Settings > Currencies で **SGD・MYR を取引通貨として追加**し、顧客ごとに通貨指定すればSGD/MYR建て請求書を作れる。試作はこれで十分。
   - 本番（澤田さん＝シンガポールでサインアップ）は基準通貨=SGDのはず。構築時に向こうの基準通貨だけ確認。
2. Preferences > Invoices：割引 **At Line Item Level**／テンプレ言語 **英語**

## 「既存生徒」ひな型（仮データ）

| 品目 | 金額(SGD) | 割引 |
|---|---|---|
| 月謝-ピアノ（1個目） | 120 | — |
| 月謝-英語（2個目） | 100 | 10% → 90 |
| テキスト代 | 30 | — |
| 合計 | 240 | |

→ Items登録 → 仮生徒作成 → Invoice作成 → PDFプレビューで形確認 → 承認後 Clone/Recurring。

## 残り3種（既存生徒が通れば早い）

- 新規生徒：入会金＋諸経費（＋初月月謝）
- 体験者：体験料 SGD35（固定＝Recurring向き）
- ホリデープログラム（春・夏）：プログラム料金（季節限定）

## 資料が来たら確定すること

- 習い事の種類と月謝額（品目名・単価の確定）
- 検定料・テキスト代・年1回諸経費の額
- 2個目割引の適用ルール（どの組合せに効くか・3個目は？）
- 生徒数（→ 無料Invoice 500件/年で足りるか、Books必要か）
- 通貨別の対象人数（SGD/JPY、MYRの開講時期）
