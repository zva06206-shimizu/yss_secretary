# 大垣精工 キーパーソン・意思決定マップ

出典：2026/06 Yutakaヒアリング。事実（本人談）と仮説を分けて記載。

## 決裁構造（事実・重要）

- **決裁権は社長に集中**。常務が大きな決裁権を持つと思いきや、**10万/月の契約すら社長決裁**だった。
- → 9月からの30万/月転換も**社長決裁**になる前提で動く。常務・推進チームを説得しても、最後は社長を通すストーリーが要る。

## 攻略の要点（クロエ・反証込み）

- **推進派（森・原）が言う本音＝「やりたいが予算・決裁が高い」**。ここがボトルネックの正体。技術や必要性の説得は不要、**社長決裁を通す材料（ROI・小さな成功事例）**を彼らに持たせるのが仕事。
- **永井氏の抵抗は技術反対ではなく運用負荷の恐れ**。潰しにかかると逆効果。設計時に「**運用が永井さん一人に降りない形**（自動化・外部運用・手順書化）」を明示し、味方化を狙う。彼はシステム管理の現場キーマンなので、敵に回すと実装が止まる。
- **社長が10万すら自分で決める** ＝ 経営者が細部まで握る会社。**社長に直接効く一枚（数字・他社事例・リスク）**を、推進派経由で届ける設計が継続の鍵。

## 外部分担（事実・重要な前提変更）

- **情報セキュリティのマネジメントシステム（ISMS）は NTT に依頼**する方針。
- → YSSの提案領域から「情報セキュリティ体制構築（ISMS）」は外れる。4/30ロードマップ中期の「情報セキュリティ強化」は**NTTとの役割分担を前提に再定義**が必要。
- ※ISO9001（品質マネジメント）とISMS（情報セキュリティ）は別物。清水のISO9001経験は引き続き有効。混同しない。
