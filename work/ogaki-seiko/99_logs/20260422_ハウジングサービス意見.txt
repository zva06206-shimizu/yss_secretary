データセンターでラックを借りてサーバを稼働させるサーバハウジングにするかどうかの方向性を決めるのはDX推進チームのメンバーの総意で決定していくようにしたいと思います。
 
また私の方からの方向性の確認が意志決定を迫るように思われたのでしたら申し訳ありません、以後気を付けたいと思います。
 
その上で、データセンターを借りてサーバ、データを集約した方が良い理由を書き出したいと思います。
この内容はDX推進チームのメンバーで方向性を決める上での判断材料として頂ければと思います。
 
－事業継続性

データ検索にかかるコストの低減
故障、過失によるデータ損失機会の低減
 
次にデータセンターにサーバを集約した際の保守性についての意見です。
 
Windows のサーバはリモートデスクトップでアクセスできるようにしますので、メンテ、操作性は問題ないと思います。
業者によるサーバの保守をする際も昨今は多くの業者がリモートアクセスツールによる一時的なリモートアクセスで作業を行っています。
例えば大塚商会は Remote Operator と言うツールを使っており、まず顧客に保守したいサーバにリモートデスクトップ接続してもらい、その状態でブラウザから大塚商会が指定するURLにアクセスし、番号を入力すると大塚商会の作業者がサーバの画面を操作できるようになり作業を遠隔地から行う事を日常的に行っています。
 
リモートデスクトップが接続できなくなった場合についてですが、サーバを購入する際にリモートから電源ON、OFF、サーバの画面表示をブラウザで表示できる機能を追加することで対応が可能です。DELL製ですと iDRAC、HPE製ですと iLO のような機能になります。
現在、大垣精工様で稼働中の「サーバ」ではこのような機能が含まれていないようですので、次回の買い換えまでは過渡期となるかも知れません。
 
機能によって社内設置する必要がある場合についてですが、私の 25年超のIT業務経験から 3D CADデータ、加工機などから出力される IoT データを保存するファイルサーバもしくはNASはデータセンターではなく社内のローカルネットワーク内に設置した方が良いようです。
 
その他、Active Directory を構成し、各拠点には Active Directory コントローラーを小さなサーバ機で設置することになるため、そのサーバ機にDHCP、DNS機能を統合することで、名前解決の対策も実施可能になります。
 
次回、私の役割とDX推進チームのみなさまの役割について、再確認させて頂ければと思います。
 
私の方としましては、DXを推進するためのソリューションの提供、業者に依頼せずともこれまでの経験で私自身が作業できる部分は手を動かして対応していければと思っております。
