# 部材表（BOM）DB化・原価分析 要件整理

> 前回2月案件で扱った「Excel部品構成ファイル（見積・原価表）」を集計・DB化する要件のマスク版。
> 具体的な部品コード・原価値を含む原文は [06_data/received/202602/仕様プロンプト_原文.txt](../06_data/received/202602/仕様プロンプト_原文.txt)（ローカル限定）。

## 背景（事実）

- カヨウは**受注の大半が三菱電機**。多くの部品が**三菱電機からの支給品**。
- 三菱電機向け製品の**原価構成（見積）を Excel「部品構成ファイル」で管理**している。
- 1ファイル=1ASSY、シート毎にSub ASSY、シートを親→子→孫の順に並べる**多階層BOM**。
- 各シートは「材料費明細／加工費明細／管理費・金利等」の3ブロック構成。列レイアウトに**2パターン（A/B）**あり、行数・工程コード行が可変。
- 別途「**年間支給部材一覧（材料支給明細）**」があり、販売品から毎月支給される部品の直入数（親図番×部品コード＝親子＋数量）を持つ。決算月9月、12ヶ月累積。

## やりたいこと（＝システム化の狙い）

1. 各部品構成ファイルの各シートを **Level 付きで再帰結合**し、BOMを串刺しで検索・分析可能にする。
   - A/Bパターンの列名を正規化して統合（材質規格=部品名、所要量=数量、金額=材料費計 など）。
   - 生産終了品（年間支給部材一覧の部品コードからたどれないもの）に**フラグ列**を付与。
2. 年間支給部材一覧の親子関係で、1)の数量を **Level 付き再帰結合**し支給実績とBOMを接続。
3. **Python で Excel → SQLite（parts_data.db）へ抽出**。入力/出力/ログのフォルダをパラメータ化。参考=既存 `extract_assy_xls_integrated.py`。
4. **Power Query から ODBC で parts_data.db に接続**し、現場が検索・分析できる状態にする。
