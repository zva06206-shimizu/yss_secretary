# BOM（部品構成ファイル）版管理の問題と分析

作成：2026-07-01

## 問題（事実・現場ヒアリング）

- 三菱電機から受領したBOMファイルが**工場長のPCに保存**されている。**共有フォルダなし**＝属人管理。
- ファイルは**800件以上**。
- **生産中止・改訂済などは工場長がファイル名や中身を見れば分かるが、明文化された規則がない**。
- そのため**我々（YSS）やシステムでは最新版・有効版の判定が困難**だった。これが前回、照合システム化が止まった一因。

## 命名規則の分析（実データ614件・`file_list*.txt` より・集計値のみ）

ファイル名は `#{部品コード}_{改訂}_{日付}.xls`（例：改訂=54B、日付=YYYYMMDD）の構造。

| 指標 | 値 | 意味 |
|---|---|---|
| 対象ファイル | 614 | 手元サンプル（実際は800超） |
| 命名規則通りに解析成功 | **610（99.3%）** | ほぼ全件が機械パース可能 |
| ユニーク部品コード | 375 | 実際の部品種類 |
| 複数版を持つ部品 | **176（46%）** | 旧版が大量に混在＝最新版管理の核 |
| 1部品の最多版数 | 5 | |
| 同一改訂・日付違い | **107部品（28%）** | 重複保存の疑い＝人手確認が要る領域 |
| 破損・規則外 | 4 | zip混入・日付欠落・#欠落・空名 |

## 何が言えるか（判断）

1. ファイル名の解析で**「版の候補を機械で並べる」ことはできる**（99.3%パース可能）。部品コードでグルーピングし改訂・日付順に整列するところまでは自動化できる。
2. **ただし「最大改訂＝有効版」は成立しない**。【重要・現場反証 2026-07-01】**機械判定で最新と思った版が、実は旧版と"並行生産"されているケースがある**。＝1部品で複数版が同時に有効になり得る。→ **ファイル名だけでは"今動いている版"を確定できない**。
3. **"今動いている版"の真の判定材料は「年間支給部材一覧（材料支給明細）」**。実際に毎月支給・生産されている部品コード（＝支給実績）が有効版の証拠。**版管理はファイル名ではなく支給実績を軸に据える**べき。
   - 支給明細にある部品コード → 有効（並行生産も支給実績で捕捉できる）。
   - 支給明細に無い → 生産中止/休眠の候補フラグ（仕様原文の「たどれないもののフラグ」と一致）。
4. **28%（同一改訂・日付違い）と破損4件、および並行生産の判別は機械では決められない**。ここは**工場長の判断 or ルール化**が要る＝「全自動」を約束してはいけない。
5. 版管理（有効版の確定）が片付くと、**BOM×支給の照合が「有効版だけ」を対象にできて格段に楽になる**＝前回の"照合困難"の主因を1つ潰せる。

## 再提案への効き方（正直な売り方）

- 打ち出し＝「**800ファイルの混沌を、9割自動整理＋例外だけ工場長が確認する"管理台帳"に変える**」。
- 過大約束を避ける：全自動ではなく「機械判定＋例外レビュー」の二層。28%の重複と中止判定にこそ現場知が要る、と明示する。
- 経営効果：属人化解消（工場長依存の脱却）・最新版取り違えによる手戻り/不良の防止・原価分析の前提データの正確化。
