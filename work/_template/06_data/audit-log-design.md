# Audit Log Design

## 目的

重要な業務操作の履歴を残し、トラブル調査、内部統制、運用改善に使える状態にする。

## 監査対象操作

| 操作ID | 対象データ | 操作 | 記録理由 | 重要度 | 保存期間 |
|---|---|---|---|---|---|
| A001 |  | 作成 / 更新 / 削除 / 承認 / 差戻し / 出力 / 連携 |  | High / Medium / Low |  |

## 標準ログ項目

| 項目 | 内容 |
|---|---|
| log_id | ログID |
| occurred_at | 発生日時 |
| actor_id | 操作者ID |
| actor_name | 操作者名 |
| action | 操作種別 |
| target_type | 対象データ種別 |
| target_id | 対象データID |
| before_value | 変更前データ |
| after_value | 変更後データ |
| ip_address | IPアドレス |
| user_agent | ユーザーエージェント |
| result | 成功 / 失敗 |
| reason | 理由、補足 |

## 注意

- パスワード、APIキー、トークンはログに保存しない。
- 個人情報をログに含める場合は、目的と閲覧権限を定義する。
- CSV出力、ファイルダウンロード、権限変更は原則として記録する。
- AIによる分類、要約、判定も監査対象に含める。
