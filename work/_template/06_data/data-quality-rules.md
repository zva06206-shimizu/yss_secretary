# Data Quality Rules

## 目的

蓄積した業務データを、検索、集計、AI分析、経営判断に使える品質で保つ。

## 品質観点

| 観点 | 説明 | 確認方法 |
|---|---|---|
| 完全性 | 必要項目が入力されている | 必須チェック、未入力一覧 |
| 一意性 | 同じデータが重複していない | 重複チェック |
| 正確性 | 実態と合っている | 承認、確認フラグ |
| 一貫性 | ステータスや分類が揃っている | マスタ管理、Enum定義 |
| 最新性 | 古い情報が放置されていない | 更新日、棚卸し |
| 追跡性 | 変更履歴を追える | 監査ログ |

## 入力品質ルール

- 必須項目を明確にする。
- 自由入力を増やしすぎない。
- 選択肢化できるものはマスタまたは選択項目にする。
- 日付、金額、数量は型と単位を固定する。
- 顧客名、担当者名、商品名などはマスタ参照を優先する。
