# Data Retention and Permission

## 目的

業務データの保存期間、閲覧権限、削除方針を定義する。

## 保存期間定義

| データ名 | 保存期間 | 保存理由 | 削除方法 | 備考 |
|---|---|---|---|---|
|  |  | 業務 / 法務 / 監査 / 分析 | 物理削除 / 論理削除 / アーカイブ |  |

## 権限定義

| データ名 | 管理者 | 承認者 | 一般利用者 | 閲覧のみ | 備考 |
|---|---|---|---|---|---|
|  | CRUD | CRU / R / None | CRU / R / None | R / None |  |

## 機密区分

| 区分 | 内容 | 例 | 取扱い |
|---|---|---|---|
| Public | 公開可能 | 公開資料 | 制限なし |
| Internal | 社内限定 | 業務マニュアル | 社内のみ |
| Confidential | 機密 | 顧客情報、見積、契約 | 権限者のみ |
| Personal | 個人情報 | 氏名、住所、連絡先 | 目的・権限・保存期間を明示 |

## 削除・アーカイブ方針

- 削除してよいデータと、履歴保持すべきデータを分ける。
- KPIや監査に使うデータは、削除時の影響を確認する。
- AI分析に使ったデータを削除する場合、派生データの扱いも決める。
- 個人情報削除時は、ログや添付ファイルも確認する。
