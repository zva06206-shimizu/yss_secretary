# KPI Definition

## 目的

業務データを、経営判断・現場改善に使えるKPIとして定義する。

## KPI一覧

| KPI ID | KPI名 | 目的 | 算出式 | データ元 | 更新頻度 | 確認者 | 目標値 | 備考 |
|---|---|---|---|---|---|---|---:|---|
| KPI001 |  |  |  |  | 日次 / 週次 / 月次 |  |  |  |

## 標準KPI候補

| 領域 | KPI | 説明 |
|---|---|---|
| 売上 | 売上高 | 対象期間の売上合計 |
| 粗利 | 粗利額 / 粗利率 | 売上から原価を引いた利益 |
| 工数 | 作業時間 | 業務・担当者・案件別の投入時間 |
| 生産性 | 1人あたり処理件数 | 人員あたりの処理量 |
| 品質 | 不良件数 / ミス件数 | 再作業や手戻りの原因分析に使う |
| 納期 | リードタイム | 着手から完了までの時間 |
| 教育 | 教育時間 | 新人・担当変更時の立ち上がり時間 |
| 定着 | 利用率 | システムに入力・確認されている割合 |
| 改善 | 改善件数 | 現場から出た改善提案数 |

## KPI設計ルール

- KPIは必ず業務データと接続する。
- 算出式を曖昧にしない。
- 手入力KPIと自動集計KPIを分ける。
- 誰が見るかを決める。
- どの会議で確認するかを決める。
- 目標値が未定の場合は未定と書く。
